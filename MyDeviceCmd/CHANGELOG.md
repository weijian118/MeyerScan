# MyDeviceCmd 变更记录

## 2026-07-24 - 1.0.0 采集命令与原始流语义入口

- 新增采集流配置、开始/停止、原始 B 包接收和结构化诊断公共接口；DeviceCmd 继续作为 CaptureService 唯一设备语义入口，不把 USB 句柄暴露给 UI 或算法 DLL。
- Capture Profile 显式传递 `queueDepth=64`、16384 字节 B 包和接收超时到 DeviceTransport；第一版不建立第二个用户态原始包队列。
- 模拟后端生成连续 0~5 六张图，并支持单次超时、连续超时、断连和部分包故障注入，供 CaptureService 无硬件回归。
- 保持“一个组六图边界最多两条无回包命令、两条之间至少 5 ms”的调度合同；自动曝光算法未实现时不生成或发送伪造参数。
- 公共 schema 升级为 `7`，整数 ABI 升级为 `8`，语义 API 升级为 `3.0.0`；代码/文件版本升级为 `1.0.0`。

## 2026-07-23 - 采集命令时序和设备上下文方案同步（未修改代码）

- 明确 `0x0A/0x0B`、曝光和灯光命令由 DeviceCmd 统一编码和发送，采集间隔内最多两条无回包命令。
- 第一条命令以 USB OUT 传输完成为当前发送成功条件，第二条命令前至少等待 5 ms；整组关灯时跳过自动曝光但仍允许发送其他合法无回包命令。
- 方案统一规定当前所有适配机型 Profile 使用 `queueDepth=64`；关灯组仍由上层完成慢速后处理和 UI 发布。本轮仅更新文档，代码落实和实机验证后续完成。
- 规定命令会话必须记录机型系列、Profile、设备编号状态、设备编号（有则记录）、设备型号/型号代码（有则记录）、生产模式和固件。
- 明确 DeviceCmd 不负责图像像素处理和自动曝光算法，自动曝光由会话级算法对象计算。
- 详细方案同步到 `F:\MeyerScan\Documents\设备相关\数据采集-原始图像预处理方案.md`。

## 2026-07-21 - 0.7.5

- 修正命令时序语义：50 ms 是上一条命令未收到普通合法帧或业务可识别终态回包时，为设备保留的下一条发送前响应窗口，不是所有命令之间的固定间隔。
